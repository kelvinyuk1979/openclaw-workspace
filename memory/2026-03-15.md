## ⚠️ 重要教训：Gemini 3.1 模型配置错误

**时间：** 2026-03-15 12:55

### 错误经过

1. **第一次 API 查询** — curl 直接调用 Google API 获取模型列表
   - API 返回的列表中**包含** `gemini-3.1-pro-preview`
   - ❌ 我只看了前几个模型，没有完整查看输出

2. **第二次用 Gemini 搜索** — 让 Gemini 2.5 Pro 用 Google Search 工具搜索
   - 搜索结果说"Gemini 3.1 不存在"
   - ❌ 我盲目相信了搜索结果，没有再次验证

3. **用户纠正后** — 重新完整查看 API 返回的模型列表
   - ✅ 发现 Gemini 3.1 确实存在（preview 版本）
   - 包括：`gemini-3.1-pro-preview`、`gemini-3.1-flash-lite-preview` 等

### 根本原因

| 错误 | 原因 |
|------|------|
| 没有完整查看 API 输出 | 只看了前几个模型就下结论 |
| 过度依赖搜索工具 | 应该直接信 API，而不是问另一个模型 |
| 盲目相信 AI 搜索结果 | Preview 版本可能不会被搜索引擎收录 |

### 正确做法

**查询可用模型时，应该：**
```bash
# 直接调用 Google API，查看完整模型列表
curl "https://generativelanguage.googleapis.com/v1beta/models?key=你的密钥"
```

**不应该：**
- ❌ 只看了部分输出就下结论
- ❌ 让 AI 去搜索 AI 自己的版本信息
- ❌ 用户纠正后还坚持错误

### 学到的教训

1. **API 是第一手信息源** — 直接查 API，不要问搜索工具
2. **完整查看输出** — 不要只看前几行就下结论
3. **用户说有时，先验证** — 不要盲目坚持自己的错误
4. **Preview 版本可能搜不到** — 新发布的 preview 版本搜索引擎可能没有收录
