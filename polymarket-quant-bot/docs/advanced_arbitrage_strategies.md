## 🧮 五大核心数学框架

### 1️⃣ 边际多面体 (Marginal Polytope)

**问题：** 简单加法失效

```
市场 A: "特朗普赢宾州？"
  YES: $0.48, NO: $0.52, Sum: $1.00 ✓

市场 B: "共和党赢宾州 5+ 点？"
  YES: $0.32, NO: $0.68, Sum: $1.00 ✓

但存在逻辑依赖：
如果共和党赢 5+ 点 → 特朗普必须赢宾州

有效结果：3 个（非 4 个）
```

**数学框架：**
```
Z = {φ(ω) : ω ∈ Ω}  # 有效收益向量
M = conv(Z)          # 边际多面体（凸包）

套利自由价格必须在 M 内，否则可被利用
```

**整数规划解：**
```
Z = {z ∈ {0,1}^I : A^T × z ≥ b}

Duke vs Cornell 例子：
- 每队 7 个证券（0-6 胜）= 14 个条件
- 2^14 = 16,384 种组合
- 但约束：不能都赢 5+ 场

约束：
Σ z(duke, 0-6) = 1
Σ z(cornell, 0-6) = 1
z(duke,5) + z(duke,6) + z(cornell,5) + z(cornell,6) ≤ 1

3 个线性约束替代 16,384 次暴力检查
```

**检测结果：**
- 17,218 个条件中 41% 存在单市场套利
- 中位错误定价：$0.60（应为$1.00，偏差 40%）

### 2️⃣ Bregman 投影 (Bregman Projection)

**问题：** 找到套利后，如何计算最优交易？

**为什么欧几里得距离失效：**
```
||μ - θ||^2

市场使用成本函数（LMSR）
$0.50→$0.60 与 $0.05→$0.15 的信息内容不同
```

**Bregman 散度：**
```
D(μ||θ) = R(μ) + C(θ) - θ·μ

对于 LMSR：
R(μ) = Σ μ_i × ln(μ_i)  # 负熵
D(μ||θ) = KL 散度
```

**最大套利利润：**
```
max_δ [min_ω (δ·φ(ω) - C(θ+δ) + C(θ))] = D(μ*||θ)

其中 μ* 是θ在 M 上的 Bregman 投影
```

**实际表现：**
- 顶级套利者年利润：$2,009,631.76
- 策略：比别人更快更准地求解 μ*

### 3️⃣ Frank-Wolfe 算法

**问题：** 直接计算 Bregman 投影不可行（M 有指数级顶点）

**Frank-Wolfe 核心：**
```
1. 从已知顶点小集合 Z_0 开始
2. 迭代 t:
   a. 在 conv(Z_{t-1}) 上求解凸优化
      μ_t = argmin_{μ∈conv(Z_{t-1})} F(μ)
   
   b. 通过解 IP 找到新下降顶点
      z_t = argmin_{z∈Z} ∇F(μ_t)·z
   
   c. 加入活跃集
      Z_t = Z_{t-1} ∪ {z_t}
   
   d. 计算收敛间隙
      g(μ_t) = ∇F(μ_t)·(μ_t - z_t)
   
   e. 如果 g(μ_t) ≤ ε 则停止
```

**Barrier Frank-Wolfe（解决梯度爆炸）：**
```
M' = (1-ε)M + εu

自适应ε递减：
如果 g(μ_t) / (-4g_u) < ε_{t-1}:
  ε_t = min{g(μ_t)/(-4g_u), ε_{t-1}/2}
否则:
  ε_t = ε_{t-1}
```

**收敛速度：**
```
O(L × diam(M) / t) → O(1/(ε×t))

实际：50-150 次迭代足够
```

**性能数据：**
| 阶段 | IP 求解时间 |
|------|-----------|
| 早期（<30 场 settled） | <1 秒 |
| 中期（30-40 场） | 10-30 秒 |
| 晚期（50+ 场） | <5 秒 |

**FWMM vs LCMM：**
- 45 场后：首次 30 分钟内完成投影
- 之后：FWMM 比 LCMM 中位提升 38%

### 4️⃣ 非原子执行约束

**问题：** CLOB 顺序执行导致风险

```
计划：
买 YES @ $0.30 ✓
买 NO @ $0.30 ✓
总成本：$0.60
保证收益：$1.00
利润：$0.40

现实：
买 YES @ $0.30 ✓
价格更新
买 NO @ $0.78 ✗
总成本：$1.08
收益：$1.00
结果：-$0.08 亏损
```

**VWAP 分析：**
```
VWAP = Σ(price_i × volume_i) / Σ(volume_i)

每 2 秒区块计算：
VWAP_yes + VWAP_no - 1.0 > 0.02 → 套利机会
```

**流动性约束：**
```
市场价差：$0.85（应有$1.00）
潜在利润：$0.15/美元
订单簿深度：$234
最大可提取利润：$234 × 0.15 = $35.10
```

**执行成功率：**
| 类型 | 检测 | 执行 | 失败原因 |
|------|------|------|----------|
| 单条件套利 | 7,051 | 87% | 流动性 48%、价格变动 31%、竞争 21% |
| 组合套利 | 13 对 | 45% | 流动性不足 71%、速度竞争 18% |

### 仓位大小逻辑

**修正凯利公式（考虑执行风险）：**
```
f = (b×p - q) / b × √p

其中：
b = 套利利润百分比
p = 完全执行概率（从订单簿深度估算）
q = 1 - p

上限：订单簿深度的 50%
```

## 🎯 关键洞察

### 零售 vs 量化

| 维度 | 零售 | 量化 |
|------|------|------|
| 检查频率 | 30 秒 | 实时 WebSocket |
| 依赖检测 | 手工/Excel | 整数规划 |
| 最优交易 | 猜测 | Bregman 投影 |
| 执行 | 手动顺序 | 并行自动 |
| 延迟 | 2.65 秒 | 30ms |
| 结果 | 提供流动性 | 提取$40M |

### 为什么跟单失败

```
Block N-1: 快系统检测 mispricing，30ms 内提交 4 笔交易
Block N: 全部确认，套利完成，你看到
Block N+1: 你跟单，但价格已从$0.30→$0.78

结果：你不是套利，是提供退出流动性
```

### 资金效率问题

```
顶级套利者：$500K+ 资本
小资金（$5K）问题：
- 滑点占小仓位比例更大
- 无法分散足够机会
- 单次失败抹去数天利润
- 固定成本（gas）占比更高

Gas 费（4 腿策略）：~$0.02
$0.08 利润 → 25% 给 gas
$0.03 利润 → 67% 给 gas

这就是为什么最低阈值是$0.05
```
