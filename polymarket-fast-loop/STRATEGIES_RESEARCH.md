**目标**: 寻找可行的自动化交易策略

## ❌ 已排除策略

```
当 YES + NO < 1 时，两边买入
```

- 活跃市场价差为 0
- 手续费 (2%) 吃掉利润
- 专业机器人竞争

**结论**: 不可行 ❌

## ✅ 待研究策略

### 策略 1: 趋势跟踪 (Trend Following)

**逻辑**:
- 当 YES 价格持续上涨 → 买入 YES
- 当 YES 价格持续下跌 → 买入 NO
- 持有到趋势反转

**优点**:
- 不需要预测事件结果
- 顺势而为，胜率较高
- 适合热门市场（大量交易推动趋势）

**缺点**:
- 需要实时价格数据
- 可能遇到假突破
- 需要止损机制

**参数建议**:
```python
LOOKBACK_PERIOD = 10      # 看过去 10 个周期
MOMENTUM_THRESHOLD = 0.05 # 5% 动量阈值
STOP_LOSS = 0.15          # 15% 止损
TAKE_PROFIT = 0.30        # 30% 止盈
```

**适用市场**:
- 高流动性市场
- 有明确趋势的市场
- 距离结算还有时间的市场

---

### 策略 2: 均值回归 (Mean Reversion)

**逻辑**:
- 当价格偏离"公平价值"过大 → 反向操作
- 赌价格会回归合理区间

**例子**:
```
某市场 YES 价格突然从 0.50 跌到 0.20
但没有新的利空消息
→ 买入 YES，赌反弹到 0.35-0.40
```

**优点**:
- 高胜率（价格通常会回归）
- 可以设置明确的入场/出场点

**缺点**:
- 需要判断"公平价值"
- 可能遇到基本面变化（不是波动而是真变了）
- 需要较快的手速

**参数建议**:
```python
DEVIATION_THRESHOLD = 0.20  # 偏离 20% 入场
TARGET_REVERSION = 0.10     # 回归 10% 出场
MAX_HOLD_TIME = 3600        # 最多持有 1 小时
```

**适用市场**:
- 流动性充足
- 无明显基本面变化
- 短期波动大的市场

---

### 策略 3: 事件驱动 (Event-Driven)

**逻辑**:
- 在重大事件发生前建仓
- 事件发生后平仓

**例子**:
```
美联储利率决议前 1 小时:
- 分析市场预期（YES 价格）
- 如果有信息优势 → 建仓
- 决议公布后 → 平仓获利
```

**优点**:
- 可以基于真实信息优势
- 潜在回报高
- 持仓时间短，风险可控

**缺点**:
- 需要信息源
- 事件时间固定，机会有限
- 可能遇到"买预期，卖事实"

**适用场景**:
- 经济数据发布
- 政治事件
- 公司公告
- 体育比赛

---

### 策略 4: 流动性提供 (Market Making)

**逻辑**:
- 同时在 YES 和 NO 两边挂单
- 赚取买卖价差 (Spread)
- 类似交易所做市商

**例子**:
```
当前市场：YES=0.50, NO=0.50

我挂单:
- YES 买单 @ 0.49, 卖单 @ 0.51
- NO 买单 @ 0.49, 卖单 @ 0.51

每次成交赚取 0.02 差价
```

**优点**:
- 市场中性，不赌方向
- 可以持续盈利
- 适合高流动性市场

**缺点**:
- 需要快速调整报价
- 遇到单边行情会亏损
- 需要较大资金

**参数建议**:
```python
SPREAD_TARGET = 0.02      # 2% 价差
MAX_POSITION = 500        # 最大持仓 500 USDC
REBALANCE_THRESHOLD = 0.05 # 偏离 5% 重新平衡
```

---

### 策略 5: 跨市场套利 (Cross-Market Arbitrage)

**逻辑**:
- 同一事件在不同平台价格不同
- 低价买入，高价卖出

**例子**:
```
Polymarket: YES = 0.48
PredictIt:  YES = 0.52

→ Polymarket 买入 YES，PredictIt 卖出 YES
→ 锁定 0.04 差价
```

**优点**:
- 理论上无风险
- 可以大额交易

**缺点**:
- 需要多个平台账户
- 资金效率低
- 平台间转账有时间风险

**适用平台**:
- Polymarket
- PredictIt
- Kalshi
- Betfair

## 🎯 推荐策略：事件驱动

**理由**:
1. **风险最低** - 基于真实信息，不是赌博
2. **回报最高** - 信息优势可以带来 10-50% 回报
3. **频率适中** - 每天 1-3 次机会，不会过度交易
4. **可自动化** - 可以监控事件日历，自动提醒

**实施步骤**:

### 第 1 步：建立事件日历
```python
EVENTS = [
    {
        'date': '2026-03-15 14:00',
        'event': '美联储利率决议',
        'markets': ['fed-rate-march-2026'],
        'expected_volatility': 'high'
    }
]
```

### 第 2 步：信息源整合
- 新闻 API（Reuters, Bloomberg）
- Twitter 监控（关键人物）
- 链上数据（大额转账）
- 社交媒体情绪

### 第 3 步：决策逻辑
```python
if 事件前 30 分钟:
    收集市场情绪
    对比预期 vs 实际
    if 有明显信息优势:
        建仓
    else:
        观望

if 事件发生后:
    立即平仓
```

### 第 4 步：风险管理
- 单笔最大投注：100 USDC
- 每日最大亏损：200 USDC
- 止损：-15%
- 止盈：+30%
